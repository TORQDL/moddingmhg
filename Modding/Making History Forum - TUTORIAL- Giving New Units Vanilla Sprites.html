TUTORIAL: Giving New Units Vanilla Sprites

Step by step on how to add a vanilla sprite to a new unit.

1.

Make the unit; This can be done either via editor, or XML, XML is a quicker way than loading up the editor, but either works. If made in the editor, notice how it uses an infantry sprite without a decal.

2, Editor.

Open up the XML file in your scenario's directory entitled "sharedDataUnitTemplateSprites.xml". Depending on what type of category your unit fits into, add it to every single area of the file. For this guide, copy and paste the entry for Advanced Tank and paste the entry below the original. Rename it to your unit.

2, XML.

If you used XML to create your unit, follow the above step, but make sure when you add your unit, you are using the line "strName" not "sidName". Notice how Basic Infantry shows up as "PreWarInfantry" in the XML file, this is because that is the "strName" of the unit. Which is used in the code.

3.

After adding your unit to "sharedDataUnitTemplateSprites.xml", go into this folder: C:\Program Files\Muzzy Lane Software\Making History Gold\Modern Europe\system\DisplayLibraries\SpriteMaps and copy every .xml file in that folder, and paste it into your DisplayLibraries folder, do not add it to another folder inside DisplayLibraries. Rename world.xml (in your scenario, not the original file) to SpriteMaps.xml.

4.

After adding the files to your scenario directory. Open up the following files in an XML EDITOR if you dont have one, get one, honestly it will make the coding 10x harder while using MS notepad (not Notepad++).

MKDTableUnitTemplatesLand.xml MKDTableUnitTemplatesLandCombat.xml MKDTableTemplateAliasesLand.xml MKDKeySet.xml

And add an entry in each of those files for your unit. The quickest way to do this is hit CTRL+F (opens up keyword finder) and type in "Advanced Tank" and it will scroll to and highlight the area where that word shows up, which shows you where you should add your entires.

5.

That's all.

To make sure the unit works, load up the editor and your scenario, and make a new army with the new unit in it. Since we are using the advanced tank model for a new armored unit (Modern Tank using Advanced Tank model, for example) it should show up on the map as an advanced tank for whichever country you gave ownership of the army.

If it doesnt work read on:

Possible Problem #1)

Scenario doesnt load in-game or in-editor when you try to test your new unit:

This is the easiest problem to solve. Start debugview (if you dont have it, again, get it, its extremely usefull for XML debugging in MH), and then startup MH. Go to the point where you have your scenario selected, and then in debugview, hit CTRL+X to clear the log, and then load up your scenario. A bunch of lines should pop up, if you dont see any usefull information, then go into each of the files you edited, and double check to make sure you didn't put something in the wrong place, or make sure spelling is correct, make sure everything is perfect, and then try again (with debugview). If debugview shows the files you edited and states an error, check that file and make sure you didnt leave any errors, or accidentally mispell something.

Possible Problem #2)

New sprite doesnt work, but scenario loads, no problems in XML either.

This is probably the most common, make sure in the "scenario.xml" file in your scenario, the line at the very bottom of the file (scroll all the way down), in the "strSpriteMapName" entry, the text "World" isnt erased:

property name="strSpriteMapName" value="World" <-- Incorrect

property name="strSpriteMapName" value="" <-- Correct

Make sure you've renamed "World.xml" in your display libraries folder to "SpriteMaps.xml" if the scenario stops loading when you make this change. If you've done all this and it still gives error code 4, look at "problem 1)" for more information.

If you're still getting problems, then honestly, i don't know what to tell you. This is a complicated process, and you probably screwed up because i am explaining it badly. It's a trial and error process. If you keep getting XML errors, then keep on loading the scenario and keep fixing them until it works. Imagine this same process, double the complication and time it takes to do, and then multiply that by 2, to get the amount of time and difficulty it took me to add the two new tank units to Road to War.

Links:

DebugView: http://technet.microsoft.com/en-us/sysinternals/bb896647

Good XML editor: XML Copy Editor: http://xml-copy-editor.sourceforge.net/

My personal favorite XML editor: Notepad++: http://notepad-plus-plus.org/

If you are willing to pay $$: XMLSpy 2011 http://www.altova.com/simpledownload2c.html?gclid=CJyd_LfsgacCFRQ8gwody3tWew
